# <a name="configure-both-routing-preference-options-for-a-virtual-machine"></a>Настройка параметров маршрутизации для виртуальной машины

В этой статье показано, как настроить параметры [маршрутизации](./routing-preference-overview.md) (в Интернете и в глобальной сети Майкрософт) для виртуальной машины. Это достигается с помощью двух виртуальных сетевых интерфейсов: одного сетевого интерфейса с общедоступным IP-адресом, направляется через глобальную сеть Майкрософт, а другой — с общедоступным IP-адресом, направленным через сеть ISP.

## <a name="prerequisites"></a>Предварительные требования

Создайте виртуальную машину с общедоступным IP-адресом, следуя инструкциям в разделе [Создание виртуальной машины со статическим общедоступным IP-адресом с помощью портал Azure](./virtual-network-deploy-static-pip-arm-portal.md) с помощью портал Azure. Выбор параметров маршрутизации по умолчанию для общедоступного IP-адреса осуществляется через **глобальную сеть Майкрософт**. После создания виртуальной машины с общедоступным IP-адресом добавьте второй общедоступный IP-адрес в виртуальную машину, которая направляет трафик через транзитную сеть ISP с предпочтениями маршрутизации, настроенными как **Интернет**.

## <a name="create-a-public-ip-address-with-a-routing-preference-choice-internet"></a>Создание общедоступного IP-адреса с выбором параметров маршрутизации в Интернете
1. Войдите на [портал Azure](https://portal.azure.com/).
2. Выберите **Создать ресурс**. 
3. В поле поиска введите *общедоступный IP-адрес*.
4. В результатах поиска выберите **Общедоступный IP-адрес**. Затем на странице **Общедоступный IP-адрес** щелкните **Создать**.
5. В диалоговом окне **Предпочтение маршрутизации** выберите **Интернет**.

      ![Создание общедоступного IP-адреса](./media/routing-preference-portal/public-ip-new.png)

    > [!NOTE]
    > Общедоступные IP-адреса создаются с адресами IPv4 или IPv6. Но сейчас при выборе предпочтительного варианта маршрутизации поддерживается только IPV4.

## <a name="create-a-network-interface-and-associate-the-public-ip"></a>Создание сетевого интерфейса и связывание общедоступного IP-адреса

1. Создайте [сетевой интерфейс](./routing-preference-overview.md) , используя портал Azure с параметрами по умолчанию. 
1. [Свяжите общедоступный IP-адрес с сетевым интерфейсом](./associate-public-ip-address-vm.md) , созданным в предыдущем разделе. Отображение IP-адреса может занять несколько секунд. Просмотрите общедоступный IP-адрес, назначенный IP-конфигурации, как показано ниже.

    ![Связывание общедоступного IP-адреса](./media/routing-preference-mixed-network-adapter-portal/public-ip.png)

## <a name="attach-network-interface-to-the-vm"></a>Подключение сетевого интерфейса к виртуальной машине

1. [Подключите сетевой интерфейс, созданный в предыдущем разделе, к виртуальной машине](./virtual-network-network-interface-vm.md).
2. Теперь можно просмотреть два виртуальных сетевых интерфейса, связанных с виртуальной машиной, одну с общедоступным IP-адресом, который направляется через глобальную сеть Майкрософт, а другой — через сеть ISP, как показано ниже:  ![ Подключение сетевого интерфейса к виртуальной машине.](./media/routing-preference-mixed-network-adapter-portal/mixed-network-adapter.png)
