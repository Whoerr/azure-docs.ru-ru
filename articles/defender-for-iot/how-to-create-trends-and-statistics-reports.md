# <a name="sensor-trends-and-statistics-reports"></a>Тенденции и статистические отчеты датчиков

Вы можете создавать графики и круговые диаграммы мини-приложений, чтобы получить представление о тенденциях и статистике сети. Мини-приложения можно группировать в пользовательских панелях мониторинга.

> [!NOTE]
> Администраторы и аналитики безопасности могут создавать отчеты о тенденциях и статистических отчетах.

Панель мониторинга состоит из мини-приложений, которые графически описывают следующие типы данных:

- Трафик по порту
- Топ трафика по порту
- Пропускная способность канала
- Общая пропускная способность
- Активное TCP-подключение
- Верхняя пропускная способность по виртуальной ЛС
- Устройства:
  - Новые устройства
  - Занятые устройства
  - Устройства по поставщикам
  - Устройства по ОС
  - Число устройств на виртуальную ЛС
  - Отключенные устройства
- Подключение отпадает по часам
- Оповещения для инцидентов по типу
- Доступ к таблице базы данных
- Мини-приложения для разсечения протокола
- делтав
  - Распределение операций Roc Делтав
  - Делтав Roc события по имени
  - События Делтав по времени
- AMS
  - AMS трафика по порту сервера
  - AMS трафика по команде
- Ethernet и IP-адрес:
  - Трафик Ethernet и IP-адресов службой CIP
  - Трафик Ethernet и IP-адреса по классу CIP
  - Трафик Ethernet и IP-адреса по командам
- СООТВЕТСТВИЕ
  - Основные операции управления OPC
  - Основные операции ввода-вывода OPC
- Компания Siemens S7:
  - S7 трафика по функциям управления
  - S7 трафика по подфункции
- Виртуальная локальная сеть
  - Число устройств на виртуальную ЛС
  - Верхняя пропускная способность по виртуальной ЛС
- 60870-5-104
  - Трафик IEC-60870 по АСДУ
- бакнет
  - Бакнет Services
- DNP3
  - DNP3 трафика по функциям
- SRTP
  - SRTP трафика по коду службы
  - SRTP ошибок по дням
- Суителинк:
  - Суителинк первые запрашиваемые Теги
  - Поведение числового тега Суителинк
- Трафик IEC-60870 по АСДУ
- DNP3 трафика по функциям
- Трафик MMS по службе
- Modbus трафика по функциям
- Трафик OPC-UA по службе

> [!NOTE]
>  Время в мини-приложениях задается в соответствии со временем датчика.

## <a name="create-reports"></a>Создавать отчеты

Для просмотра панелей мониторинга и мини-приложений:

Выберите **тренды & статистики** в боковом меню.

:::image type="content" source="media/how-to-generate-reports/investigation-screenshot.png" alt-text="Снимок экрана для расследования.":::

По умолчанию результаты обнаружения отображаются за последние семь дней. Этот диапазон можно изменить с помощью средств фильтрации. Например, поиск в свободной тексте.

## <a name="create-a-dashboard"></a>Создание панели мониторинга

Создайте новую панель мониторинга, выбрав раскрывающееся меню **панели мониторинга** . Вы можете создавать и добавлять на панель мониторинга столько мини-приложений.

Настраиваемые панели мониторинга можно создавать с помощью следующих параметров.

- Добавление мини-приложения на панель мониторинга

- Удаление мини-приложения с панели мониторинга

- Изменение фильтра мини-приложения

- Изменение размера мини-приложения

- Изменение расположения мини-приложения

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/pin-a-dashboard.png" alt-text="Изменение расположения мини-приложения.":::

Чтобы создать настраиваемую панель мониторинга, выполните следующие действия.

1. На левой панели выберите **тенденции и статистические данные** .

1. Выберите раскрывающееся меню **Выбор панели мониторинга** и нажмите кнопку **создать панель мониторинга** .

1. Введите понятное имя для новой панели мониторинга и нажмите кнопку **создать**.

1. Выберите **Добавить мини** -приложение в верхней левой части страницы.

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/widget-store.png" alt-text="Выберите мини-приложение из хранилища мини-приложений.":::

1. **Безопасность** и **рабочие** мини-приложения доступны в правом верхнем углу окна. Выбор из различных категорий и протоколов. Рядом с каждым мини-приложением отображается краткое описание с изображением миниатюры. Используйте полосу прокрутки, чтобы просмотреть все доступные мини-приложения.

1. Выберите мини-приложение, нажав **кнопку "добавить** ". Мини-приложение немедленно отображается на панели мониторинга.

Чтобы удалить панель мониторинга, выполните следующие действия.

1. Выберите имя панели мониторинга в раскрывающемся меню.

1. Щелкните значок **Удалить** , а затем нажмите кнопку **ОК**.

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/garbage-icon.png" alt-text="Щелкните значок Удалить, чтобы удалить панель мониторинга.":::

Чтобы изменить имя панели мониторинга, сделайте следующее:

1. Выберите имя панели мониторинга в раскрывающемся меню.

1. Щелкните значок **изменить** .
  
  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/edit-name.png" alt-text="Щелкните значок изменить, чтобы изменить имя панели мониторинга.":::

1. Введите новое имя для панели мониторинга и нажмите кнопку **сохранить**.
 
Чтобы задать панель мониторинга по умолчанию, сделайте следующее:

1. Выберите имя панели мониторинга в раскрывающемся меню.

1. Щелкните значок **звездочки** , чтобы выбрать панель мониторинга, которая должна быть задана в качестве панели мониторинга по умолчанию.

   :::image type="content" source="media/how-to-create-trends-and-statistics-reports/default-dashboard.png" alt-text="Щелкните значок звездочки, чтобы выбрать панель мониторинга по умолчанию."::: 

Чтобы изменить фильтрацию данных в мини-приложении, сделайте следующее:

1. Выберите значок **фильтра** .

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/filter-widget.png" alt-text="Щелкните значок фильтра, чтобы задать параметры для мини-приложения.":::

1. Измените необходимые параметры.

1. Щелкните **ОК**.

Чтобы удалить мини-приложение, сделайте следующее:

- Выберите :::image type="icon" source="media/how-to-create-trends-and-statistics-reports/x-icon.png" border="false"::: значок.

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/delete-widget.png" alt-text="Выберите X, чтобы удалить мини-приложение.":::

## <a name="creating-widgets"></a>Создание мини-приложений 

Хранилище мини-приложений позволяет выбирать мини-приложения по категориям и протоколам. Можно отобразить **Безопасность** или **рабочие** мини-приложения, выбрав их.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/widget-store.png" alt-text="Выберите мини-приложение из хранилища мини-приложений.":::

Каждое мини-приложение содержит определенные сведения о сетевом трафике, сетевой статистике, статистике протокола, устройстве и предупреждении. При отсутствии данных для мини-приложения отображается сообщение.

Вы можете удалить раздел из круговой диаграммы, чтобы более четко увидеть относительную значимость оставшихся срезов. Чтобы сделать это, выберите имя среза в условных обозначениях в нижней части экрана.

В следующих разделах представлены примеры вариантов использования для некоторых мини-приложений.

### <a name="busy-devices-widget"></a>Мини-приложение "занятые устройства"

В этом мини-приложении перечислены пять самых загруженных устройств. В режиме **редактирования** можно выполнять фильтрацию по известным протоколам.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/busy-device.png" alt-text="Представление мини-приложения занятого устройства.":::

### <a name="total-bandwidth-widget"></a>Мини-приложение общей пропускной способности

Это мини-приложение отслеживает пропускную способность в Мбит/с (мегабит в секунду). Пропускная способность указывается на оси y с датой, отображаемой на оси x. Режим **редактирования** позволяет фильтровать результаты в соответствии с клиентом, сервером, портом сервера или подсетью. При наведении курсора на диаграмму появляется подсказка.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/total-bandwidth.png" alt-text="Представление мини-приложения &quot;Общая пропускная способность&quot;.":::

### <a name="channels-bandwidth-widget"></a>Мини-приложение пропускной способности каналов

В этом мини-приложении отображаются пять ведущих каналов трафика. Можно выполнить фильтрацию по адресу и задать число отображаемых результатов. Щелкните стрелку вниз, чтобы отобразить больше каналов.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/channels-bandwidth.png" alt-text="Представление мини-приложения пропускной способности каналов.":::

### <a name="traffic-by-port-widget"></a>Трафик по мини-приложению порта

Это мини-приложение отображает трафик по порту, который обозначается круговой диаграммой с каждым портом, назначенным другим цветом. Объем трафика в каждом порте пропорционален размеру его части круга.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/traffic-by-port.png" alt-text="Представление трафика по порту.":::

### <a name="new-devices-widget"></a>Мини-приложение "новые устройства"

В этом мини-приложении отображается линейчатая диаграмма новые устройства, показывающая, сколько новых устройств было обнаружено в определенной дате.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/new-devices.png" alt-text="Представление мини-приложения &quot;новые устройства&quot;.":::

### <a name="protocol-dissection-widgets"></a>Мини-приложения для разсечения протокола

В этом мини-приложении отображается круговая диаграмма, позволяющая взглянуть на трафик по протоколу, вводятся по кодам функций и службам. Размер каждого среза пропорциональен объему трафика относительно других срезов.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/protocol-dissection.png" alt-text="Представление мини-приложения &quot;раздел&quot; протокола.":::

### <a name="active-tcp-connections-widget"></a>Мини-приложение активных подключений TCP

В этом мини-приложении отображается диаграмма, показывающая количество активных TCP-подключений в системе.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/active-tcp.png" alt-text="Представление активного мини-приложения TCP-подключений.":::

### <a name="incident-by-type-widget"></a>Мини-приложение по типу инцидента

В этом мини-приложении отображается круговая диаграмма, показывающая количество инцидентов по типу. Число предупреждений, созданных каждым ядром за определенный период времени.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/incident-by-type.png" alt-text="Представление мини-приложения типа инцидент по типу.":::

## <a name="devices-by-vendor-widget"></a>Устройства по мини-приложению поставщика

В этом мини-приложении отображается круговая диаграмма, показывающая количество устройств по поставщикам. Число устройств для конкретного поставщика пропорционально размеру его части, относящейся к устройству, относительно других поставщиков устройств.

## <a name="number-of-devices-per-vlan-widget"></a>Число устройств на мини-приложение виртуальной ЛС

В этом мини-приложении отображается круговая диаграмма, показывающая число обнаруженных устройств на виртуальную ЛС. Размер каждого среза пропорционален количеству обнаруженных устройств относительно других срезов.

Каждая виртуальная ЛС отображается с тегом VLAN, назначенным датчиком или именем, добавленным вручную.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/number-of-devices.png" alt-text="Представление мини-приложения число устройств.":::

### <a name="top-bandwidth-by-vlan-widget"></a>Верхняя пропускная способность по мини-приложению VLAN

Это мини-приложение отображает использование пропускной способности виртуальной ЛС. По умолчанию мини-приложение отображает пять виртуальных ЛС с наибольшим использованием пропускной способности.

Данные можно отфильтровать по периоду, представленному в мини-приложении. Щелкните стрелку вниз, чтобы отобразить дополнительные результаты.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/top-bandwidth.png" alt-text="Представление верхней пропускной способности мини-приложения VLAN.":::

## <a name="system-report"></a>Системный отчет

Чтобы скачать системный отчет, выполните следующие действия. 

1. Выберите **тренды & статистики** в боковом меню.

1. Выберите **системный отчет** в правом верхнем углу. Отчет будет скачан автоматически.

  :::image type="content" source="media/how-to-create-trends-and-statistics-reports/system-report.png" alt-text="Нажмите кнопку системный отчет, чтобы скачать копию системного отчета.":::

Системный отчет представляет собой PDF-файл, содержащий все данные в системе:

  - Устройства

  - видны узлы

  - Сведения о сетевой политике

## <a name="devices-in-a-system-report"></a>Устройства в системном отчете 

В системном отчете отображается список всех устройств и сведения о них. Например, введите, имя и используемые протоколы. В системном отчете также отображается список устройств для каждого поставщика.

## <a name="alerts-in-system-report"></a>Оповещения в системном отчете 

В системном отчете отображается список всех предупреждений со сведениями, такими как Дата и серьезность.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/alerts-report.png" alt-text="Представление предупреждений в системных отчетах.":::

## <a name="network-information-in-system-report"></a>Сведения о сети в системном отчете 

В системном отчете подробно описаны базовые показатели сети. Например, код функции DNP3 и открытые порты на подключение.

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/dnp3.png" alt-text="Представление функции DNP3 из системного отчета.":::

:::image type="content" source="media/how-to-create-trends-and-statistics-reports/open-port.png" alt-text="Представление отчета о открытом порте для соединения.":::
